Week 1

The main objective of Week 1 was to compare the received components against our listed inventory to ensure all parts were present and functional. Upon inspection, all components were present except for the magnets. The project was divided into two main sections: hardware and software. The hardware section was further divided into two more sections: power generation and gps & accelerometer.

Software

For the software section, we installed the necessary libraries for interfacing with the GPS and accelerometer modules.

A mock test with fake drivers was conducted to ensure the Raspeberry Pi Pico would be able to generate data, store the data in memory and if the data would later be accessible for analysis.

Hardware

For the hardware section, we began by setting up the power generation module. This involved using the concept of Faraday torch(shake flashlight) to generate power through motion. A copper coil was wrapped around a PVC pipe and a magnet was attached to the inside of the pipe. When the pipe is shaken, the magnet moves within the coil, generating electricity. However, due to the absence of the magnets, the power generation module was not fully functional.

Next, we focused on the GPS. The GPS module was connected to the Raspberry Pi Pico using UART communication to track the location of the hockey stick. A GPS patch antenna was also attached to the module to improve signal reception. The set up was placed near window to improve signal strength. However, due to the lack of a proper GPS signal in the indoor environment, we were unable to fully test the GPS functionality.
